-ADO significa ActiveX Data Objects . Es un camino por el cual la aplicación puede acceder a los datos que reside en una base de datos . 
Aquí la base de datos es un término general; que incluye una gran variedad de bases de datos específicas - excel hojas , archivos de texto , 
SQL Server , base de datos Jet , etc. En la terminología orientada a objetos , ADO es simplemente un modelo de objetos que nos permite interactuar
 con una base de datos para que podamos buscar, añadir o modificar datos . En el nivel superior tenemos un objeto Connection . Este objeto nos permite
 establecer una conexión con una base de datos . La mejor parte que realmente no tiene que preocuparse por los detalles acerca de la base de datos subyacente. 
ADO es compatible con una gran colección de bases de datos como se mencionó anteriormente . En el nivel siguiente viene el objeto Command y el objeto Recordset . 
El objeto Command de ADO se utiliza para ejecutar consultas contra una base de datos. Si va a recuperar información de una base de datos que tiene que guardarlo 
en un objeto Recordset . Supongo que esto va a hacer por la breve introducción para ADO
